DevOps-учебник: доска объявлений

15 уровней от одного контейнера до self-healing production-стека. Каждый уровень решает боль, которую создал предыдущий: строим → ломаем → наблюдаем → чиним.
Как пользоваться. Эта страница — карта курса со всеми схемами. Кликай уровень → читай «боль → решение» → открывай диаграммы (каждая — со своей темой и экспортом в PNG/SVG). Команды и полные шаги — в README внутри папки уровня в репозитории. Разметка: [голова] — концепция, боль→решение, теория: читается с телефона в дороге; [руки] — README уровня и практика: нужна VM.

0Настройка VPSlevel-0-vps-setup/
ssh-ключи · ufw · fail2ban · unattended-upgrades
БОЛЬ
Свежий сервер с root-паролём — боты начинают брутфорс через минуты после появления IP в сети.
РЕШЕНИЕ
Ключи вместо паролей, Default Deny на firewall, непривилегированный пользователь, fail2ban.
концепцияслои защиты реализацияреальный hardening сетьпуть SSH-подключения
1Монолитlevel-1-monolith/
docker compose · nginx · fastapi · postgresql · jwt · alembic
БОЛЬ
Ничего нет. Стартуем с самой простой рабочей конфигурации — и ломаем её нагрузкой (k6), чтобы почувствовать пределы.
РЕШЕНИЕ
Первый живой сервис: статика через nginx, API, база с миграциями, авторизация. Плюс бэкап с restore drill.
концепция3-tier реализацияреальный compose сетьпуть запроса
2Горизонтальное масштабированиеlevel-2-scaling/
nginx upstream · 3× backend · least_conn
БОЛЬ
Один бэкенд задыхается под нагрузкой: latency растёт, появляются ошибки.
РЕШЕНИЕ
3 инстанса за балансировщиком + stateless-архитектура (JWT) — убийство контейнера под нагрузкой переживается без потерь.
боль→решение концепциябалансировка реализацияupstream сеть
3Кэшированиеlevel-3-caching/
redis · cache hit/miss · инвалидация · индексы
БОЛЬ
Три бэкенда упираются в одну базу: одинаковые SELECT-ы молотят PostgreSQL на каждый запрос.
РЕШЕНИЕ
Redis перед базой: HIT — миллисекунды, MISS — идём в базу и кладём в кэш; инвалидация при записи.
боль→решение концепцияcache-aside реализация
3.5HTTPSlevel-3.5-https/
traefik · let's encrypt · acme · redirect http→https
БОЛЬ
Всё ходит по HTTP: пароли и JWT видны любому на пути (Wi-Fi, провайдер).
РЕШЕНИЕ
Traefik на границе терминирует TLS и сам получает/продлевает сертификаты Let's Encrypt.
боль→решение концепцияtls на границе реализацияtraefik сеть
4CI/CDlevel-4-gitlab/ (+ level-4-cicd/ — GitHub Actions)
gitlab ce · runner · registry · rules · trivy · hadolint
БОЛЬ
Деплой руками: ssh, git pull, docker restart — сервис падает на время деплоя, шаги забываются.
РЕШЕНИЕ
Pipeline: тесты → сборка образа (тег = git SHA) → скан уязвимостей → rolling deploy по кнопке.
боль→решение концепцияpipeline реализация.gitlab-ci.yml сеть
5Kuberneteslevel-5-kubernetes/
minikube · deployment · service · probes · hpa · blue-green · canary
БОЛЬ
Compose не перезапускает упавшие контейнеры, не масштабирует под нагрузкой, не переживает ребут сервера.
РЕШЕНИЕ
Декларируешь желаемое состояние — reconciliation loop сам чинит и масштабирует (self-healing, HPA).
боль→решение концепцияжелаемое состояние реализацияманифесты сетьnodeport→pod
5.5Ingress и cert-managerlevel-5.5-ingress/
ingress-nginx · clusterip · cert-manager · acme · clusterissuer
БОЛЬ
Наружу торчит NodePort :30080 — нестандартный порт, без TLS, по порту на каждый будущий сервис.
РЕШЕНИЕ
Единый вход :80/:443 с маршрутизацией по Host; cert-manager сам выпускает и продлевает сертификаты.
боль→решение концепцияправила vs контроллер сетьпуть через ingress теорияacme http-01
6Observabilitylevel-6-monitoring/
prometheus · alertmanager · grafana · loki · promtail · cadvisor
БОЛЬ
Инцидент — а ты слепой: не видно RPS, latency, ошибок; логи размазаны по контейнерам.
РЕШЕНИЕ
Метрики (Prometheus) + дашборды (Grafana) + централизованные логи (Loki) + алерты (Alertmanager).
боль→решение концепция3 столпа реализациястек в compose сеть
6.5AI-агент диагностикиlevel-6.5-ai-agent/
python · claude api · alertmanager webhook · telegram · wireguard
БОЛЬ
Алерт в 3 ночи: 50 000 строк логов, гипотезы, 30 минут до причины — вручную.
РЕШЕНИЕ
Webhook от Alertmanager → агент сам собирает логи и метрики → LLM объясняет и предлагает действие (approve/reject в Telegram).
боль→решение концепцияагент в контуре реализацияsequence сетьvpn + egress
7Helmlevel-7-helm/
chart · values · required · rollback · blue-green · canary
БОЛЬ
Деплой — десяток kubectl apply по очереди; «что именно было задеплоено вчера?» — неизвестно.
РЕШЕНИЕ
Один чарт = весь стек одной командой; история ревизий, rollback, разные values для dev/prod.
боль→решение концепцияblue-green реализациячарт
8GitOpslevel-8-gitops/
argocd · selfheal · prune · drift detection · immutable tags
БОЛЬ
Кто-то поправил кластер руками — Git и реальность разошлись, никто не заметил (drift).
РЕШЕНИЕ
Git — единственный источник правды: ArgoCD непрерывно сверяет кластер с репо и откатывает самодеятельность.
боль→решение концепцияpush vs pull реализацияapplication + :latest ловушка
8.5Секреты в GitOpslevel-8.5-secrets/
sealed secrets · kubeseal · sops/age · external secrets (обзор)
БОЛЬ
GitOps требует всё состояние в Git — но secret.yml в Git это утечка, а base64 не шифрование.
РЕШЕНИЕ
В Git — шифротекст (SealedSecret); расшифровать может только контроллер с приватным ключом внутри кластера.
боль→решение концепцияпуть секрета
9Terraformlevel-9-terraform/
hcl · state · plan/apply · variables · environments
БОЛЬ
Инфраструктура создана кликами и руками — staging «такой же, но немного другой», воспроизвести нельзя.
РЕШЕНИЕ
Инфраструктура как код: описал в .tf → plan показывает diff → apply приводит к состоянию. Идемпотентно.
боль→решение концепцияstate реализация
10Ansiblelevel-10-ansible/
inventory · playbooks · roles · handlers · vault
БОЛЬ
Terraform создал 10 VM — а настраивать каждую (пакеты, конфиги, сервисы) опять руками?
РЕШЕНИЕ
Плейбуки по SSH на весь парк сразу: идемпотентные task-и, роли, handlers, секреты в Vault.
боль→решение концепцияpush по ssh реализацияплейбук сетьfan-out
